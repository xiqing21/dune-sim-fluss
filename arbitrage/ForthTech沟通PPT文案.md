## 第 6 页：Delta Join——零状态关联

**对应演讲**：数据流转线 Delta Join 部分

**内容**：

**传统方案**：双流 Join → TB 级 RocksDB 状态 → 检查点 90 秒

**Fluss 方案**：Delta Join → 零状态 → 检查点 1 秒

**淘宝实测数据**：
- Join 状态：100TB → 0
- 检查点时间：90s → 1s
- 资源消耗：2300 CU → 200 CU（降低 10 倍）

**量化场景意义**：链上 Swap + CEX 深度 + Chainlink 验证，一次 Join 完成，无需 TB 级状态

## 第 8 页：Union Read——回测与实盘零偏差

**对应演讲**：策略验证部分

**内容**：

**传统痛点**：Lambda 架构下回测偏差率 20-30%

**Fluss + Paimon 方案**：

- 回测：Flink Batch 从 Paimon 读历史
- 实盘：同一 Flink SQL 流模式从 Fluss 读实时
- **同一套代码，同一个数据源，流批偏差归零**

```
SELECT * FROM dex_swaps  -- Fluss 热数据(6h) + Paimon 冷数据(全量)
WHERE token = 'RAVE'
  AND ts BETWEEN '2026-01-01' AND '2026-03-31'
-- Union Read 自动合并，一个 SQL 获取完整历史
```

## 第 12 页：三层架构——极速 + 实时 + 历史

**对应演讲**：分层架构部分

**内容**：

```
极速决策层（< 50ms）
  CEX WebSocket → 内存规则引擎（C++/Rust）→ CCXT API → 交易所
  不经过 Fluss，直接内存处理
  专门服务：HFT、低延迟做市

实时分析层（< 500ms）
  CEX WebSocket → Fluss → Flink → 信号/风控
  Delta Join · CEP · 窗口计算
  专门服务：中性套利、Funding、链上Alpha

历史分析层（秒级-分钟级）
  Fluss → Tiering → Paimon → Trino/StarRocks
  全量历史 · 策略回测 · 绩效归因
```

**三层共享 Fluss 统一数据总线，渐进式升级**
**技术栈对齐**：ForthTech 用 Python/C++/Rust，极速层天然匹配
