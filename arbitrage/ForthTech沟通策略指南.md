# ForthTech 沟通策略指南

## 定位、话术、节奏、风险应对

---

## 一、你的核心定位

### 1.1 不要说"我只有架构和设想"

这是你最大的心理障碍，但实际上它不成立。ForthTech 最缺的不是"会写合约的人"或"会跑策略的人"，而是**能设计可靠数据基础设施的架构师**。

换个角度看你的能力：

| 你以为的短板 | 实际上的优势 |
|-------------|-------------|
| 没有实操 Web3 经验 | 有 5 年国家电网级企业数仓实战，这是 ForthTech 最稀缺的"机构级稳定"基因 |
| 没有量化实操经验 | 已经完成 Spread 套利策略原型（RAVE 案例拆解、PNL 计算、风控设计），比 90% 的纯技术人更懂业务 |
| 只有架构设想 | 架构设想 + 6 份深度研究报告 + 竞品分析 + 经济效益量化，比很多"有实操经验"的人更系统 |
| Vibecoding 能力 | AI 辅助快速原型开发，在 Web3 这个快速迭代的赛道是真正的生产力倍增器 |

### 1.2 你应该怎么定位自己

**推荐定位：数据架构师 / 量化 Infra Lead**

不要说"我想来学习 Web3 开发"，而要说"我带来了国网级别的数据架构能力，帮助你们把数据基础设施从策略层融合升级到数据层融合"。

核心叙事线：**"我在地球上最复杂的数据环境（国网）积累了 5 年数仓实战，现在要把这个能力迁移到 Web3 量化场景。你们有策略和交易经验，我有数据架构和实时计算能力，正好互补。"**

### 1.3 身份角色选择

根据对话进展，灵活选择：

**最佳情况——高级数据架构师 / 量化 Infra Lead**：负责 Fluss + Flink + Paimon 数据基础设施的设计和搭建，让策略团队专注策略逻辑。这是你最匹配的角色。

**次优情况——技术顾问 / 联合开发**：先以项目制合作，帮助搭建 CEX-DEX 私有数据库集成模块。门槛最低，容易先落地小胜。

**备选——CTO/技术合伙人级别的架构设计**：如果你能证明架构价值，可以谈更高的角色。

**不要选的角色**：普通 Web3 开发、前端工程师、纯后端——这些角色会暴露你无 Web3 开发经验的短板。

## 二、沟通节奏和话术

### 2.1 开场（微信跟进）

保持专业低姿态，不要一次发太多信息。核心是：**一个钩子 + 一个明确请求**。

推荐话术：

```
Darren 总，您好！

感谢之前在 Web3 嘉年华的交流，您分享的量化基础设施思路让我很有共鸣。

我之前在国网负责大数据中台架构 5 年，最近在研究 Web3 场景下的实时数据融合方案。我做了一个 CEX + DEX + 预言机多源实时集成的架构设计，和 ForthTech 的跨所套利业务场景高度匹配。

整理了一份简报，方便的话想占用 15-20 分钟跟您汇报一下，看看是否有合作机会？
```

**注意**：不要在消息里展开技术细节，目标是约到一次 15-20 分钟的沟通。

### 2.2 15-20 分钟沟通的结构

```
第 1-3 分钟：我是谁 + 为什么找你
  - 5 年国网大数据数仓架构
  - 3 年币圈自驱研究
  - 在 Web3 嘉年华听到你们的分享，发现高度匹配

第 3-10 分钟：我看到什么问题 + 我的解决方案
  - 问题：多源数据融合在当前 Web3 量化场景的痛点
  - 方案：FlinkCDC → Fluss → Flink → Paimon 三线融合架构
  - 重点讲：数据线（多源融合）+ 资金线（风控闭环）+ 策略线（信号驱动执行）
  - 用 ForthTech 的业务场景举例（Spread 套利、Funding 收割、DEX 套利）

第 10-15 分钟：市场验证 + 差异化
  - 市场空白：没有平台完整融合 DEX + CEX + 预言机做量化
  - 竞品对比：Amberdata 有技术没交易、Nansen 有产品没 CEX、FMZ 有执行没融合
  - Goldsky 的验证：Redpanda + Flink 架构，云成本节约 3-4 倍

第 15-20 分钟：下一步
  - 请求：是否可以就 CEX-DEX 私有数据集成模块做一次更深入的技术讨论
  - 或者：是否有兴趣看一个 15 分钟的技术 Demo
```

### 2.3 核心话术要点

**讲问题时用数据说话**：
- "arXiv 2025 年的研究，仅 19 个搜索者就从 720 万次 CEX-DEX 套利中提取了 2.338 亿美元"
- "Aave 在 2025 年 2 月单日清算量达 2 亿美元，如果有实时健康因子监控，可以帮用户避免 5% 以上的清算损失"
- "2024-2025 年预言机操纵攻击造成 DeFi 损失超过 4 亿美元"

**讲方案时用场景说话**：
- "你们做 Spread 套利，当前每个策略需要自己处理 CEX 深度数据、计算价差、监控 Funding、跟踪保证金率——我的方案是把这些下沉到数据层，策略只管开平仓逻辑"
- "你们有链上大户追踪需求，当前链上数据（Dune）和 CEX 数据是割裂的，我的方案是一个 SQL 同时查询链上大户转账和 CEX 充值记录"

**讲优势时用对比说话**：
- "Goldsky 用 Redpanda + Flink 架构，云成本节约 3-4 倍——Fluss 比 Redpanda 更进一步，原生支持主键更新、列裁剪、Delta Join"
- "Dune 的 Sim API 和 DuneSQL 是两套独立系统，无法跨 Join——我的 Fluss + Paimon Union Read 是一个 SQL 自动合并实时和历史"

## 三、关于学历和经验的应对

### 3.1 绝对不要主动提

**学历和外包身份绝对不要主动提起**。对方没有问就不要说。这不是欺骗，而是沟通策略——你要让对方先看到你的价值，再决定是否在意你的背景。

### 3.2 如果对方问起学历

如果对方明确问到学历，用"能力 + 成果"来回应：

```
"我的学历是成人本科在读，但我认为在技术领域，能解决问题比学历更重要。
我在国网 5 年，从零搭建了 TB 级实时数仓，这是实打实的工程经验。
过去半年我独立完成了 6 份 Web3 数据架构的深度研究报告，
覆盖竞品分析、架构设计、场景探索和经济效益量化，
这些成果可以证明我的学习能力和专业深度。"
```

**核心原则**：承认但不展开，快速转到能力和成果。

### 3.3 如果对方问起 Web3 开发经验

```
"我没有传统意义上的 Web3 开发经验（没写过智能合约、没发过币），
但我有 Vibecoding 能力——能通过 AI 辅助快速原型开发。
更重要的是，我做的是数据架构层，不需要写合约，
需要的是 Flink、Fluss、Paimon 这些大数据技术栈，
这正是我 5 年国网经验的核心能力。"
```

### 3.4 如果对方问起实时经验

```
"我的实时计算经验主要来自研究和架构设计，目前还没有大规模生产级实践。
但我已经完成了 FlinkCDC → Fluss → Flink → Paimon 全链路的架构设计，
包括 Delta Join、CEP、窗口计算等核心场景的方案设计。
理论深度已经足够，需要的是一个生产环境来验证和迭代。"
```

## 四、关于办公地点的应对

### 4.1 不要在初次沟通中提

办公地点是入职后的问题，不是初次沟通的重点。如果对方没有提到，你不要主动问。

### 4.2 如果对方问起

```
"我目前在惠州，但完全可以灵活安排。
如果是远程协作，我之前在国网也有远程开发的经验；
如果需要到香港，我可以定期出差。
具体方式可以根据项目需要灵活调整。"
```

**底线**：不要因为地点问题在初次沟通中就把自己限死。等对方认可你的价值后，地点问题通常可以协商。

## 五、关于另一份远程大数据工作的处理

### 5.1 绝对不要在 ForthTech 沟通中提另一份工作

这会让你显得不够专注和有诚意。在 ForthTech 的沟通中，你的态度应该是"我选择了 ForthTech 而不是其他机会"。

### 5.2 时间线管理

- ForthTech 沟通：今天发消息 → 24-48h 内等回复 → 安排 15-20 分钟电话
- 远程大数据面试：同步推进，不要等 ForthTech 回复才去面试
- 如果两边都有 offer：再根据薪资、发展空间、兴趣做选择

### 5.3 如果 ForthTech 进展慢

7 天后礼貌再追一次。同时在 X/LinkedIn 公开发布你的 Web3 数据架构研究摘录，制造二次曝光。即使 ForthTech 没成，这些公开内容也是你进入香港 Web3 量化圈的敲门砖。

## 六、沟通禁忌

**不要做的事**：

1. 不要说"我只有架构和设想，没有实操经验"——说"我完成了从 0 到 1 的架构设计，现在需要一个生产环境来验证"
2. 不要说"我不知道 Fluss 够不够快"——说"Fluss 的延迟在亚毫秒级，对于 Spread 套利场景完全适配，极速路径可以绕过 Fluss 直连交易所"
3. 不要说"我是成人本科还没毕业"——除非对方直接问
4. 不要说"我还有另一份面试在等"——这会让你显得不够专注
5. 不要在技术讨论中暴露不确定性——你做了 6 份研究报告，你对这个架构的理解比 95% 的人都深
6. 不要一上来就谈薪资和职位——先让对方认可你的价值
7. 不要在微信消息里展开技术细节——目标是约到沟通机会

**要做的事**：

1. 用数据和场景说话，而不是泛泛而谈
2. 把你的研究成果转化为对方能理解的业务语言
3. 保持专业低姿态，但不要自我贬低
4. 准备好 15 分钟 Demo 提纲，以备对方要求
5. 跟进时简洁有力，不要长篇大论

## 七、最佳结果和最差结果

**最佳结果（概率 30-40%）**：安排 15-30 分钟电话/见面 → 对方认可架构价值 → 邀请你做技术深度讨论或小项目试水 → 逐步转为全职/顾问

**次优结果（概率 30%）**：对方给了反馈，让你补充材料 → 你继续迭代 → 转为顾问/合作

**保底结果（概率 20-30%）**：暂时没回复或对方暂时没有需求 → 这仍然是进入香港 Web3 量化圈的绝佳敲门砖 → 7 天后礼貌再追一次 → 同时继续推进远程大数据面试

**关键心态**：你是在"选择"而不是"求职"。你有国网 5 年经验 + 系统性 Web3 架构研究 + 策略原型，这在市场上是稀缺的。即使 ForthTech 没成，你的研究成果也足以敲开其他 Web3 量化团队的大门。
