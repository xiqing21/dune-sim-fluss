# 研究计划: 国网 AI+大数据平台设计 — 基于 Dune/Sim MCP 与 Skill 架构分析

## 研究目标
参考 Dune 和 Sim 的 MCP 权限控制模型与 CLI Skill 设计模式，为国家电网（国网）业务设计一套 AI+大数据平台的架构方案，形成完整技术文档。

## 研究维度

### 维度1: Dune/Sim MCP 权限控制与 Skill 设计深度分析
**研究任务**:
- Dune MCP 工具的权限分级体系（公开查询 vs 私有查询、信用点计费、执行权限）
- Sim API 的密钥/访问控制模型（API Key、CU 计费、多级权限）
- CLI Skill 的设计模式（命令映射、参数校验、错误处理、结果格式化）
- MCP 工具分层设计原则（只读 vs 写入、单次 vs 批量、同步 vs 异步）

**关键信息点**:
- Dune 工具权限矩阵: searchTables(公开) → createDuneQuery(需认证) → executeQueryById(需认证+信用点)
- Sim 工具权限: 实时查询需 API Key，按 CU 计费
- Skill 命令设计: dune query run-sql, dune dataset search, dune sim evm balances
- 权限与计费的解耦设计

### 维度2: 国网大数据业务场景与需求分析
**研究任务**:
- 国网核心业务数据域: 电力调度、设备监测、用电采集、营销服务、安全监控
- 实时数据处理需求: 电网运行监测、故障预警、负荷预测
- AI 应用场景: 设备缺陷识别、负荷预测、异常检测、智能巡检
- 数据安全与合规要求: 等保三级、数据分级分类、内网隔离

**关键信息点**:
- 国网数据平台现状: 数据中台、SG-UAP、数据湖
- 电力行业大数据政策: 电力数据要素市场化
- 国网 AI 平台: 智脑、AI 中台

### 维度3: 国网 AI+大数据平台架构设计
**研究任务**:
- 参照 Dune/Sim MCP 分层设计国网数据访问工具层
- 设计国网特色的权限控制模型（角色+数据域+操作级别）
- 设计国网大数据 Skill 体系（调度、监测、分析、预测）
- 实时流处理架构选型（Flink/Fluss 在电力场景的应用）
- AI 能力集成方案（大模型+专业模型混合架构）

**设计原则**:
- 数据安全第一: 内网部署、数据不出域
- 实时性优先: 电网调度秒级响应
- 模块化设计: Skill 可独立升级和扩展
- 渐进式开放: 从只读分析到预测决策
